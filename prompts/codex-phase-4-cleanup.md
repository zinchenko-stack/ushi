# Задача: Phase 4 — финальная уборка перед релизом

Тебе передают macOS-приложение **ushi** после успешно завершённых Phase 2 (бандл whisper-cli, коммиты `bc62b04`, `c08f19c`), Phase 3 (онбординг скачивания модели, `1e117e1`) и Phase 3.5 (фоновое скачивание, `4e7f344`). Эта задача — финальная уборка перед тем как давать DMG друзьям/жене.

---

## 1. Что такое ushi (краткая выжимка)

Личный macOS-рекордер для созвонов и лекций. SwiftUI, target macOS 14.6+. На данный момент полностью работающий пайплайн:
- Запись системного звука + микрофона + опционально видео
- Бандленный whisper-cli (статический, ARM64, с Metal)
- Модель `ggml-large-v3-turbo` (1.5 ГБ) скачивается на первом запуске через onboarding
- Можно свернуть онбординг и пользоваться приложением (запись), пока модель качается
- После загрузки модели накопленные `.pending` записи транскрибируются последовательно
- Распространение через DMG без подписи Apple
- Подробнее в `CLAUDE.md`

## 3. Пошагово

### Шаг 1. Обновить `INSTALL.md`

Открой `INSTALL.md` в корне репозитория. Сейчас он описывает старую модель (`brew install whisper-cpp`, ручное скачивание модели). Нужно переписать раздел установки и разрешений под новый flow.

Структура нового `INSTALL.md`:
```markdown
# Установка ushi

## Первый запуск
1. Скачай ushi.dmg → перетащи приложение в Applications
2. Запусти Ushi из Программ. Mac заблокирует — «не удалось проверить разработчика».
3. **Системные настройки → Конфиденциальность и безопасность** → внизу
   «Открыть всё равно» → введи пароль → подтверди «Открыть»

### Подготовка приложения (один раз)
При первом запуске Ushi скачает модель распознавания речи (~1.5 ГБ).
Окно «Подготовка Ushi» покажет прогресс. Можно «Свернуть и пользоваться сейчас» —
записи будут работать, транскрипция начнётся когда модель догрузится.

## Разрешения
- **Запись экрана** — без неё не записывается системный звук.
- **Микрофон** — для записи твоего голоса. Если выключишь микрофон в Ushi,
  это разрешение не нужно.

После выдачи прав Ushi автоматически перезапустится.

## Обновления
Приложение само проверяет наличие новой версии при запуске. Внизу появится
плашка «Доступна версия X · Скачать». После скачивания нового DMG: закрой
текущую версию, перетащи новую в Applications с заменой, запусти.

## Если что-то сломалось
- **Не открывается:** проверь, что в Конфиденциальность → Запись экрана для
  Ushi включена галка.
- **Не пишется звук собеседника:** Запись экрана отключена либо требует
  повторной выдачи после обновления macOS.
- **Не слышно твой голос:** проверь переключатель «Микрофон» в Ushi.
- **Модель не качается:** проверь интернет, нажми «Повторить» на онбординге
  или баннере. Модель скачивается с huggingface.co.
- **Транскрипция падает с ошибкой:** скорее всего модель скачалась
  битая — удали `~/.ushi/models/ggml-large-v3-turbo.bin` и перезапусти Ushi,
  скачивание начнётся заново.

## Поддерживаемые версии macOS
- macOS 14.6 Sonoma и новее.
- Захват микрофона: macOS 15 Sequoia и новее.
```

**Важно:** удали полностью упоминания brew, ручного скачивания модели, ручного запуска `whisper-cli`. Этого больше нет.

### Шаг 4. Опционально: SHA-256 валидация модели

В Phase 3 мы отложили валидацию хеша. Сейчас можно добавить — это защитит от:
- Битого скачивания (модель повреждена в процессе)
- Подмены модели (теоретически)

Возьми актуальный sha256 модели с HuggingFace:
```bash
curl -sL "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3-turbo.bin?download=true" -I | grep ETag
# или скачай через `git lfs` и сделай `shasum -a 256`
```

Хеш скорее всего: `1fc70f774d38eb169993ac391eea357ef47c88757ef72ee5943879b7e8e2bc69`
(но **проверь сам перед коммитом** — может обновиться когда они обновят модель).

Добавь в `ModelManager`:
```swift
private let expectedModelSHA256 = "1fc70f774d38eb169993ac391eea357ef47c88757ef72ee5943879b7e8e2bc69"

private func verifyModel(at url: URL) async throws -> Bool {
    let data = try Data(contentsOf: url, options: .mappedIfSafe)
    let hash = SHA256.hash(data: data)
    let hashStr = hash.compactMap { String(format: "%02x", $0) }.joined()
    return hashStr == expectedModelSHA256
}
```

После завершения скачивания (но **до** атомарного перемещения в финальное место):
1. Посчитай sha256 скачанного файла
2. Если не совпадает с expected → удалить, выставить `.failed("Файл повреждён, повторите скачивание")`, дать пользователю кнопку «Повторить»

`import CryptoKit` понадобится.

**Если решишь отложить** — оставь как есть, эта часть не блокирующая. Но **добавь TODO** в `ModelManager` с пометкой `// TODO(phase 5): SHA-256 validation`.

### Шаг 5. Финальный сквозной тест

Перед коммитом прогони себя сам как «свежий пользователь»:

```bash
# 1. Удалить модель
rm -f ~/.ushi/models/ggml-large-v3-turbo.bin*

# 2. Собрать релиз
./scripts/build-release-dmg.sh

# 3. Смонтировать собранный DMG
open release/build/ushi.dmg

# 4. Перетащить ushi.app в /Applications (если уже есть — заменить)

# 5. Запустить из Applications
open /Applications/ushi.app
```

Прогони сценарий:
- Должен открыться онбординг → «Подготовка Ushi» с прогрессом
- Жми «Свернуть и пользоваться сейчас» → должна появиться плашка в основном окне
- Сделай 5-секундную запись → она должна сохраниться со статусом «ждёт загрузки модели»
- Дождись завершения скачивания → плашка исчезает → запись начинает транскрибироваться
- Проверь что итоговый текст появился

Если всё ок — Phase 4 готова.
